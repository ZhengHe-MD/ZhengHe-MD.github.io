meta refresh 可以当作客户端 301 用

把博客从 /blog/YYYY/MM/DD/slug/ 迁到 /writing/slug/ 时，旧链接不能就这么断掉。GitHub Pages 是纯静态托管，给不了服务端 301，但 meta refresh 加上 canonical 就够用了：浏览器立刻跳转，搜索引擎则顺着 canonical 把权重归到新地址。

转发桩由索引器从每个页面自带的 legacy 元数据自动生成，49 篇一个不落，不用手写。

<meta http-equiv="refresh" content="0; url=/writing/go-project-layout/">
<link rel="canonical" href="https://zhenghe-md.github.io/writing/go-project-layout/">
